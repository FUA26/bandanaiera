# Image Upload Enhancement - Release Notes v1.1.0

**Release Date:** 2025-04-05

## Overview

This release enhances the image upload system across the backoffice with crop, compression, and multiple image support. Services now have image upload capability, and all forms use consistent UI patterns.

## Features Added

### Core Features
- **Enhanced Image Upload Component** (`EnhancedImageUploader`)
  - Drag & drop file upload
  - Client-side image compression (max 1MB, 1920px width)
  - Image cropping modal with aspect ratio presets
  - Multiple image support (up to 10 images)
  - Progress tracking and error handling
  - Image gallery with drag-to-reorder

### Form Integration
- **Services Form**: Now supports multiple image uploads
- **News Form**: Enhanced with crop and compression
- **Events Form**: Enhanced with crop and compression
- **Tourism Form**: Enhanced with crop and compression

### API Updates
- Public services API now includes images
- Admin services API handles imageIds
- File upload endpoint integrated with MinIO

### Landing Page
- Service cards now display uploaded images
- Fallback to icon when no images available
- Optimized image display with hover effects

## UI Improvements

### Consistent UI Patterns
- **1-7 fields**: Dialog/popup (Categories, Tags)
- **8+ fields**: Full page (Services, News, Events, Tourism)

### Service Form Changes
- Removed `ServiceDialog` component
- Services now use full page form only
- Better for complex forms with 25+ fields

### User Experience
- Real-time upload progress feedback
- Improved error messages (Indonesian language)
- Better visual feedback during drag & drop
- Toast notifications for success/error states

## Performance

### Client-Side Optimizations
- Images compressed before upload (max 1MB, 1920px, quality 0.85)
- Format conversion: PNG → WEBP for files > 2MB
- Reduced bandwidth usage
- Faster upload times

### Server-Side
- Images stored in MinIO with CDN URLs
- Efficient database queries with Prisma
- Public API cached (300s TTL)

## Database Changes

### Schema Updates
```prisma
model Service {
  // ... existing fields

  images  File[]   @relation("ServiceImages")
  imageIds String[] @default([])
}
```

## Known Issues

### Current Limitations
- Maximum 10 images per service
- Maximum 5MB file size before compression
- Image cropping requires CSS import
- Compression not supported in very old browsers

### Minor Issues
- Image reordering in gallery (drag-to-reorder) has visual feedback but needs polish
- Preview generation can be slow for very large files
- Reference count updates for deleted files may need optimization

### TypeScript Errors (Pre-existing)
The type checking revealed several pre-existing TypeScript errors unrelated to this release:
- Next.js route handler parameter type changes (params as Promise)
- Missing dependencies (@radix-ui/react-collapsible, vaul)
- Chart component type issues
- Event/News service type issues

These errors do not affect the image upload functionality and should be addressed in a separate cleanup task.
